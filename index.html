THE ARCHITECTURE

Agents reach out.
The firewall stays shut.

Every other remote-access tool asks you to open a hole toward the machine. Filebox inverts the direction: the agent opens one long-lived outbound WebSocket. Your browser only ever talks to the hub.

Browser → Hub: standard HTTPS Agent → Hub: outbound WSS only Firewall: untouched, no inbound rules
PREVIEW, INLINE

See the figure.
Not the file path to it.

Whether it's a results plot, a correlation matrix, a cluster scatter, or a tidy data table — Filebox renders it inline, on the server where it lives, the moment you click. No scp, no jupyter, no shared drive, no syncing a laptop.

trend.pngimage
Trend lines. Two series over time — compare runs, spot regressions, judge whether the latest change helped, all without leaving the browser.
correlation.pngimage
Heatmaps & matrices. Correlation, confusion, or any grid of values — scan the structure at a glance instead of parsing a CSV.
clusters.pngimage
Scatter & clusters. Projections, distributions, or raw point clouds — rendered where they were generated, colored the way you saved them.
results.csvcsv
Tables & logs. CSV / TSV renders as a clean aligned table, log files stream in color. Open two files side by side in two tabs.

Made for people who live in data.

Researchers, analysts, and data scientists with files scattered across shared servers — preview the figure, read the dataset, check the box, move on. Never write, never execute, never proxy. Your data stays where it was written.

The death of scp

Stop copying a heavy file to your laptop just to glance at one plot or scan one dataset. The figure renders where it lives — on the server, through the browser. Your bandwidth and your disk thank you. So does your collaborator on the other coast.

Your data is safe

The protocol is read-only — there is no write, delete, rename, or exec message anywhere. A collaborator can browse a shared directory but can never clobber a dataset, corrupt a result, or drop a rogue script. Sensitive files stay denied.

5 min self-hosted

From download to browsing in five minutes.

Pre-built static musl binaries. No Rust toolchain, no Node, no Docker. Just a binary and a config file.

1
Grab the release tarball
Hub on your exposed machine, agent on each backend.
2
Generate config
./scripts/gen_config.sh hub > hub.json — only needs openssl.
3
Run the hub
./bin/hub — serves the UI on :3000.
4
Point agents outward
Each agent reads hub = "https://..." and dials home. Add roots from the UI.
zsh — hub 
$ tar xzf filebox-hub-*-x86_64-musl.tar.gz
$ ./scripts/gen_config.sh hub > hub.json
# → generates bcrypt hashes via openssl + mkpasswd
#   no Python, no Node, no external deps

$ ./bin/hub

  ✓ serving frontend dist (1.4 MB)
  ✓ agent WS endpoint ready

$ FILEBOX_AGENT_HUB=https://hub.example.com \
