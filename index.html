Architecture

Agents reach out. The firewall stays shut.

Your browser talks only to the hub. Every agent opens one long-lived outbound WebSocket. Nothing inbound is required on the machines that hold the data.

01 · Client

Browser

HTTPS from anywhere. Auth, browse, preview, stats.

02 · Control

Hub

One endpoint. Routing, sessions, live progress.

03 · Data

Agents

Dial home from HPC nodes, data boxes, edge hosts.

Read-only by protocol — no write, delete, rename, or exec. Sensitive paths are denied by default.

Capabilities

Built for glancing at remote data.

Peek at figures, scan datasets, search trees, watch live load — without copying files to a laptop.

01

Inline preview

Images, PDF, Markdown, CSV, HTML, and Monaco-powered code — rendered where the file lives.

02

Workspace search

Filename and content search scoped to a root or folder, with progress, cancel, and denylist-aware scanning.

03

Virtual collections

Named lists of file references across roots — gather a working set without moving data.

04

Multi-tab workspace

Open several previews, jump between them, navigate the directory with the keyboard.

05

System monitor

CPU, memory, load, and top processes — tuned for large HPC boxes with tens of thousands of PIDs.

06

Self-update

--update pulls the matching release, verifies checksums, and replaces the install in place.

Install

Download. Init. Run.

Static musl binaries. No toolchain, no Docker. Config is generated by the binaries themselves.

01
Grab the releaseHub tarball on the exposed host; agent on each backend.
02
Generate config./bin/hub --init-config then ./agent --init-config.
03
Start the hub./bin/hub serves the UI on :3000.
04
Point agents outPaste the hub URL and token. Add roots from the UI.
hub · zsh
$ tar xzf filebox-hub-*-x86_64-musl.tar.gz
$ cd filebox-hub-*
$ ./bin/hub --init-config
# writes config/hub.json · prints agent token once
